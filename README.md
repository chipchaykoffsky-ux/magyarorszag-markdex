# Magyarország MarkDex — Landing

Современный одностраничный лендинг на чистом HTML/CSS/JS. Готов к загрузке на любой обычный HTTP-хостинг по FTP/SFTP.

---

## 📁 Структура проекта

```
site/
├── index.html              ← главная страница
├── css/
│   └── style.css           ← все стили (тёмная тема + неон)
├── js/
│   └── main.js             ← валидация, анимации, отправка формы
├── php/
│   ├── send.php            ← обработчик формы (Вариант 2)
│   ├── .htaccess           ← защита CSV от публичного доступа

├── assets/
│   └── images/
│       └── favicon.svg
├── robots.txt
└── README.md
```

## 🚀 1) Как открыть локально

**Способ 1 — просто двойной клик** по `index.html`. Откроется в браузере.
Работает почти всё, но PHP-форма так не заработает (нужен сервер).

**Способ 2 — локальный сервер** (рекомендуется для проверки формы):

```bash
# Если установлен Python 3:
cd site
python -m http.server 8000
# затем открыть http://localhost:8000

# Или через PHP (для проверки формы):
cd site
php -S localhost:8000
# форма будет работать через php/send.php
```

## ✏️ 2) Как заменить тексты и картинки

### Тексты
Все тексты лежат прямо в `index.html`. Откройте файл в любом редакторе (VS Code, Sublime, Notepad++) и просто редактируйте содержимое между тегами.

Ключевые места:
- `<h1 class="hero__title">` — главный заголовок
- `<p class="hero__subtitle">` — подзаголовок
- Блоки `.card h3` / `.card p` — преимущества
- `<details class="accordion__item">` — FAQ

### Картинки / логотип
- Логотип — SVG, прямо в HTML (ищи `<svg class="logo__mark">`). Цвета настраиваются через `linearGradient#lg1`.
- Favicon — `assets/images/favicon.svg`
- Чтобы добавить скриншот платформы / фото — положите файл в `assets/images/` и подключите:
  ```html
  <img src="assets/images/your-image.jpg" alt="Описание" loading="lazy">
  ```

### Цвета темы
В `css/style.css` в самом верху — блок `:root` с переменными:
```css
--neon-1: #00E5FF;   /* основной неон (голубой) */
--neon-2: #7C5CFF;   /* второй неон (фиолетовый) */
--neon-3: #14F195;   /* зелёный акцент */
--bg:     #0a0b1e;   /* фон */
```
Меняйте здесь — применится везде.

## 📩 3) Как подключить форму

В файле `js/main.js` в самом верху:

```js
const FORMSPREE_URL = '';                 // Вариант 1
const PHP_ENDPOINT  = 'php/send.php';     // Вариант 2
```

### Вариант 1 — Formspree (без сервера, работает везде)

Подходит, если у вас **статический хостинг** (GitHub Pages, Netlify, shared без PHP).

1. Зарегистрируйтесь на [formspree.io](https://formspree.io) (есть бесплатный план).
2. Создайте новую форму — получите URL вида `https://formspree.io/f/xyzabc123`.
3. Откройте `js/main.js` и вставьте URL:
   ```js
   const FORMSPREE_URL = 'https://formspree.io/f/xyzabc123';
   ```
4. Готово. Если `FORMSPREE_URL` не пустой — используется он, PHP игнорируется.

Альтернативы: [getform.io](https://getform.io), [web3forms.com](https://web3forms.com), [formsubmit.co](https://formsubmit.co). Все работают по такому же принципу.

### Вариант 2 — PHP-обработчик (shared hosting)

Подходит, если у вас **обычный хостинг с PHP** (Beget, Reg.ru, TimeWeb, Hostinger и т.п.).

1. Оставьте `FORMSPREE_URL = ''` в `js/main.js`.
2. Откройте `php/send.php` и отредактируйте:
   ```php
   $TO_EMAIL   = 'kapcsolat@pelda.hu';   // ВАША почта для уведомлений
   $FROM_EMAIL = 'noreply@pelda.hu';     // отправитель (обычно почта вашего домена)
   $SUBJECT    = 'Új regisztráció — Magyarország MarkDex';
   ```
3. При желании отключите CSV-сохранение: `$SAVE_CSV = false;`
4. Загрузите сайт на хостинг (см. раздел 4).

**Что PHP-обработчик делает:**
- Валидирует поля на сервере
- Защита от ботов (honeypot + rate-limit)
- Отправляет письмо через `mail()`
- Сохраняет заявки в `php/submissions.csv` (защищён от публичного доступа через `.htaccess`)
- Возвращает JSON-ответ фронтенду

**Если `mail()` не работает** на вашем хостинге — это частая история на shared. Используйте Вариант 1 (Formspree).

## 🌐 4) Как загрузить сайт на FTP/SFTP-сервер

### Через FileZilla (самый простой способ)

1. Скачайте [FileZilla](https://filezilla-project.org/).
2. **Файл → Менеджер сайтов → Новый сайт.**
3. Заполните данные от хостинга:
   - **Хост:** `ftp.ваш-домен.hu` (или IP)
   - **Протокол:** SFTP — если доступен (безопаснее), иначе FTP
   - **Пользователь / Пароль:** из панели хостинга
4. Подключитесь.
5. На удалённой стороне перейдите в папку сайта — обычно `public_html/`, `www/`, `htdocs/` (зависит от хостинга).
6. Выделите **содержимое папки `site/`** (не саму папку!) и перетащите на сервер.
7. Откройте `https://ваш-домен.hu` — должен открыться лендинг.

### Через командную строку (rsync / scp)

Если есть SSH-доступ:
```bash
# Синхронизация (рекомендуется):
rsync -avz --delete ./site/ user@ваш-домен.hu:/var/www/html/

# Или простая заливка:
scp -r ./site/* user@ваш-домен.hu:/var/www/html/
```

### Права на файлы

После загрузки убедитесь (через FTP-клиент или `chmod`):
- HTML, CSS, JS, PHP: `644`
- Папки: `755`
- `php/` должна быть доступна на запись, если вы хотите использовать CSV-сохранение

## ❓ Проблемы?

- **Форма не отправляется локально** → нужен сервер (см. раздел 1, способ 2).
- **Шрифты не грузятся** → проверьте интернет / CSP.
- **PHP даёт 500** → проверьте в панели хостинга error_log и путь к `mail()`.
- **CSV не создаётся** → дайте папке `php/` права `755` и убедитесь, что PHP может писать.
